## Principios de Trabajo

- Progreso real > teoría
- Prototipos > documentación extensa
- Simplicidad > complejidad innecesaria
- Construir primero, optimizar después
- 3 servicios / 2 DBs / múltiples colas = primero versión más simple

---

## Forma de Trabajar

1. Entender impacto de negocio antes de escribir código
2. Análisis sistemático: reproduce, aísla, valida con datos
3. Separar por capas: handlers → services → models → database
4. MVPs funcionales antes de optimizar
5. Observabilidad: si no puedes medirlo, no puedes mejorarlo

## Riesgos a Evitar

- Burnout de cierres mensuales
- Perfeccionismo en scope (sobre-diseñar antes de validar)
- Multifoco sin prioridad clara
- Dependencia de validación externa
- Trading automatizado (descartado - alto riesgo, bajo ROI)
